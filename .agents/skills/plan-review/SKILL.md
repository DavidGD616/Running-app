# Plan Review

Review plan output using this order:

1. Scope boundary check (files, modules, APIs).
2. Risk and missing assumptions.
3. Verification path (tests/docs checks).
4. Rollback/stop conditions.

Report only concrete blockers and required edits to the plan.
