# Plan Protocol

Use for tasks that require a pre-work plan before edits.

- Include objective, scope, risks, and acceptance criteria.
- Mark whether the plan is implementation-oriented or review/documentation-only.
- Add gating conditions for commit and verification.
- Keep plan tasks atomic and task-sized.
- Explicitly call out assumptions that require user confirmation.
